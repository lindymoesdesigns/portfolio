Regulating Feeding Time

26 January 2020

Chow-E is a prototype design for a pet owner communication app. In order for animals to maintain a healthy diet, guardians should be aware of how much their pet is eating. ChowE helps pet owners communicate and track feeding times.

Food please!

Chow-E users have multiple people living in their house, own a pet, and have a busy schedule. With so much to keep track of, it is hard for them to know who has fed their pet and when.

Did you feed the cat?

Some pets have a hard time knowing when to stop eating which leaves it up to the owner to regulate feeding time. With multiple people in the house it's hard to track when pets ate especially when they eat fast and beg for more. This can cause the pet to be fed multiple times which causes overeating.​

Notify, Input, Communicate

I sat down with the product owner and did a "How Might We" exercise on features for the application. We decided to move forward with the above marked features. These would characterize the MVP (Minimum Viable Product). All others features would be prioritized for later releases.

Accessibility

A lot of brainpower went into ironing out the features in the wireframe phase so I wouldn't need to answer those questions later. My goal was to make the features slim and lightweight for the user. I realized that being able to record food intake quickly was going to be an important feature.

The most important feature for the user is to add food to a pet. This would be where the user is spending the most amount of time in the app. We decided to focus on making that experience easy and accessible for the user so it doesn't feel cumbersome over time. A tear sheet would appear from the bottom and allow the user to choose from food they've entered previously or add something new.

After doing some high fidelity mock ups I user tested the prototype and received some very valuable feedback.

Learnings

  • The user wanted the food to be separate for each pet
  • The date is redundant and taking up too much space
  • The user wants to invite a friend to feed a pet on specific days that the user has chosen

Slimming Down

From the feedback I received I consolidated the information on the Pets page and shaved off all the parts of the design that were unnecessary. The date at the top of the Pets page was taken out and I gave more room for pet information.

Future

In conclusion, I have learned a lot throughout this process of development. I was able to tap into my roots on principles of design and apply them to this app. For example, the date was not a priority for the user and in the end, it was deprioritized. User testing never fails to be a productive experience and is a practice I plan to keep in my design process.

I look forward to continuing my work on this application design. If you are interested in learning more about this and other products I am working on, stay tuned for my upcoming blog. I will be taking an in-depth and honest journey with user experience design on a new and exciting project I think you will enjoy.
